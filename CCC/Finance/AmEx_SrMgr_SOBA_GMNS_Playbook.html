S3 · 55-Day T−25/T+30 Framework
Turning ambiguity into a proprietary operational methodology
Maps to: REQ-03 (Ambiguity → Structure) · REQ-05 (Revenue Acceleration / Sales cycle reimagination)
SITUATION
Collections had no framework for identifying when a customer payment was at highest risk. Decisions about collection intensity were made on intuition, not data. RM scheduling, escalation triggers, and resource allocation were ad hoc — causing customer friction at non-critical windows and underperformance at the highest-leverage moments.
TASK
Define the problem from first principles, structure an analytical framework, and build a scalable solution — with no precedent, no academic model, and no external benchmark.
ACTION
Analysed 4 years of payment data to identify the critical collection window. Discovered that the 55-day T−25/T+30 window around payment due dates captures the highest-leverage collection opportunity. Built the framework as the operational backbone — RM scheduling, escalation triggers, customer touchpoint sequences, and documentation requirements all anchored to this window. Embedded it into SOPs, trained all 77 staff, and built the monitoring layer into the MI ecosystem for real-time adherence visibility.
RESULT
Proprietary framework now governs all collection activity timing. Portfolio delinquency reduced 74% (16.7% → 4.4%) over 4 years. The framework is original intellectual property — not adapted from any external model. SOBA reframe: "I built the sales-cycle methodology from scratch — the same 'reimagine the process' capability this JD asks for. Different domain, identical structural problem."
